### 应用层开发模板

  - 基于 koa 的 应用模板

#### 目录结构
```
.
|————bin  --------------------- 可执行脚本目录
| |————test.sh ---------------- 测试环境执行脚本
|————config ------------------- 配置文件目录
| |————config.js  ------------- build后生成，不同的环境内容不同，不用提交
| |————config_default.js  ----- 所有环境的默认配置项
| |————config_prd.js  --------- 生产环境的默认配置
| |————index.js --------------- 配置项合并逻辑，(生产环境会去读取 
|————common ------------------- 公共代码
| |————util.js  --------------- 常规工具类函数
| |————mysql.js  -------------- mysql连接初始化
| |————redis.js  -------------- redis连接初始化
| |————log.js ----------------- 日志模块初始化
|————controller --------------- 解析用户的输入，控制层目录
| |————status.js  ------------- 健康检查，服务器启动，数据库连接等
| |————example.js  ------------ 业务api样例
|————service ------------------ 编写业务逻辑层，逻辑层目录
| |————status.js  ------------- 健康检查，服务器启动，数据库连接等
| |————example.js  ------------ 业务api样例
|————model  ------------------- 数据模型层
| |————index.js --------------- 整合模型方法
| |————example.js ------------- sequelize模型样例
| |————check.js --------------- 检查mysql连接
|————middleware --------------- 自定义中间件目录
| |————auth.js ---------------- 身份认证层
| |————index.js --------------- 中间件集合
| |————log.js ----------------- 日志中间件
| |————param_valid.js --------- 参数校验层
| |————response_format.js ----- 应用层统一返回结构
|————router ------------------- 路由目录
| |————index.js --------------- 路由加载
| |————example.js ------------- 路由定义样例
|————CHANGELOG.md ------------- 项目发布版本历史记录
|————README.md  --------------- 项目描述文件
|————.eslintrc  --------------- eslint静态检查配置文件
|————.eslintignore  ----------- eslint静态检查忽略的文件
|————.gitignore --------------- 排除上传git的文件
|————yarn-lock.json  ---------- yarn包版本锁定文件, 确保每个环境安装的版本一致
|————package.json ------------- npm包依赖配置
|————app.js ------------------- 程序启动入口文件
```
